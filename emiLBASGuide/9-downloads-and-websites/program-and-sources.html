  • Ollama
  • Vllm
  • LMStudio ( windows only )
  • KoboldCPP

Notes:

  • llama.cpp is the mainstream
  • ik_llama.cpp is a fork usually has better cpu performance but "lags behind" in some other applications
  • ollama has it's own distribution , and is "simpler" ( If your tarded )
  • vllm has better support for multigpu and "serving a lot of ppl ( 10+ ) in 1 box"
  • Lm Studio has lower performance (like ollama), but good/easy for testing models.

llama.cpp -jinja "fix" for qwen3.6

  • https://huggingface.co/froggeric/Qwen-Fixed-Chat-Templates/tree/main download the chat_template.jinja
  • add --jinja --chat-template-file chat_template.jinja ^ to llama.cpp launch parameters

MCP websites :

  • Official MCP servers repo — reference implementations + community list, maintained by Anthropic/community
  • mcp.so — searchable directory, biggest catalog right now
  • Smithery — directory + one-click install/config generator, many are hosted (remote) not just local
  • Glama MCP directory — has quality/safety scoring per server
  • PulseMCP — directory + newsletter tracking new servers
  • Composio — hosted MCP integrations for real SaaS tools (Gmail, Slack, Notion, etc.), auth handled for you

Remote only

  • Zapier : server with several MCP connections , a LOT of apps , and selection of access level
  • OttoKit : hosted MCP server with 1,000+ app integrations
  • n8n: open-source workflow automation tool that exposes your custom-built automations as MCP tools for your AI via webhooks.
